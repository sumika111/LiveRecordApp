# 設計変更履歴

設計書（`BASIC_DESIGN.md`）やスキーマ（`DB_SCHEMA.md`）の変更を記録します。  
**設計を変えたときは、ここに日付・内容・理由を追記してください。**

---

## 記録のルール

- **日付** … 変更を決めた日（YYYY-MM-DD）
- **対象** … どのドキュメント／どの部分か（例: 基本設計 4. コア機能、DB_SCHEMA venues）
- **内容** … 何をどう変えたか（Before → After が分かるとよい）
- **理由** … なぜ変えたか（任意だが書いておくと後で思い出しやすい）
