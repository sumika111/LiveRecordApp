## 2. 認証まわり

- **ログイン / 新規登録** … `/login`（メール・パスワード）
- **ログアウト** … ヘッダーの「ログアウト」ボタン
- **未ログイン時** … 記録する・マイ記録・ランキングは `/login` にリダイレクト
- **Supabase 未設定時** … 500 にしないように `getOptionalUser()` でエラーを握りつぶし、ログイン画面では「環境変数を設定してください」と表示

**主なファイル:**  
`app/login/page.tsx` / `app/auth/callback/route.ts` / `lib/supabase/server.ts`（`getOptionalUser`） / `lib/supabase/middleware.ts` / `components/LogoutButton.tsx`

## 3. 画面一覧

| URL | 内容 |
|-----|------|
| `/` | ホーム。ログイン中は「記録する」「マイ記録」「ランキング」「ニックネーム設定」へのリンク。未ログインは「ログイン」リンク。 |
| `/login` | ログイン・新規登録フォーム。入力欄の文字は黒で表示。 |
| `/record` | **記録する**。会場検索 or 会場新規登録 → 公演日・**公演名**・**アーティスト名**・メモ → 保存。保存後は `/my` へ遷移。 |
| `/my` | **マイ記録**。最近10件の記録＋「カレンダーで見る」「自分だけのランキング」へのボタン。全件は `/my/list` へ。 |
| `/my/calendar` | **カレンダー**。月表示で公演があった日をマーク。日付クリックでその日の公演一覧＋**詳細**ボタン（編集ページへ）。前月・翌月で移動。 |
| `/my/list` | **全件一覧**。行った公演を日付順（新しい順/古い順）で全件表示。各件に「編集」リンク。 |
| `/my/ranking` | **自分だけのランキング**。公演数・会場数・アーティスト別回数。全体/月別/年別タブ、期間選択。 |

| `/profile` | **ニックネーム設定**。ランキングに表示する名前を設定。メールは表示されない。 |
| `/ranking` | **ランキング**。公演数・会場数・都道府県数のトップ20（ユーザー別）。表示名はニックネーム。「**月別・年別ランキングを見る**」で期間別表示へ。 |
| `/ranking/period` | **ランキング（月別・年別）**。タブで「月別」「年別」を切り替え。月または年を選ぶとその期間の公演数・会場数・都道府県数ランキングを表示（直近12ヶ月・直近3年から選択）。 |

## 4. 主なコンポーネント・処理

- **`components/RecordForm.tsx`** … 記録する画面のフォーム。会場検索・会場登録・**公演日・公演名・アーティスト名**・メモ・送信。  
  - 送信時: 同じ「会場・日付・公演名」の `events` があればそれに紐づけ、なければ新規作成（`artist_name` 含む）→ `attendances` に 1 件追加。
- **`app/my/page.tsx`** … マイ記録トップ。最近10件の記録＋「カレンダーで見る」「自分だけのランキング」へのボタン。全件は `/my/list` へ。
- **`app/my/ranking/page.tsx`** … 自分だけのランキング（公演数・会場数・アーティスト別回数、全体/月別/年別・期間選択）。
- **`app/my/list/page.tsx`** … 全件一覧。日付順（新しい順/古い順）で全件表示、各件に編集リンク。
- **`app/my/calendar/page.tsx`** ＋ **`components/MyCalendar.tsx`** … カレンダー表示。月単位で表示し、公演があった日をマーク。日付クリックでその日の公演一覧と詳細（編集リンク）を表示。
- **`app/my/edit/page.tsx`** ＋ **`components/EditEventForm.tsx`** … 公演の編集。公演日・公演名・アーティスト名・メモを更新。会場は読取専用。
- **`app/profile/page.tsx`** ＋ **`components/NicknameForm.tsx`** … ニックネーム設定。ランキングでメールを出さない。
- **`lib/ranking.ts`** … ランキング集計（公演数・会場数・都道府県数をユーザー別に集計、各トップ20）。**`getRankingsByPeriod(supabase, { year, month? })`** で月別・年別の期間指定集計。
- **`app/ranking/page.tsx`** … ランキング画面。3カラムで「公演数」「会場数」「都道府県数」を表示。「月別・年別ランキングを見る」リンクあり。
- **`app/ranking/period/page.tsx`** … 月別・年別ランキング。タブで月別/年別を切り替え、期間を選ぶとその期間のランキングを表示。

## 5. Supabase のテーブル（あなたが SQL で作成したもの）

- **profiles** … ユーザー表示名・ニックネーム（サインアップ時にトリガーで自動作成。ランキングではニックネーム表示）
- **venues** … 会場（名前・都道府県・市区町村）。重複は「同じ名前＋都道府県」で防止
- **events** … 公演（会場・日付・公演名・**アーティスト名**・メモ）。「会場＋日付＋公演名」で一意。編集可能（RLS: 参加している公演のみ更新可）
- **attendances** … 参加記録（誰がどの公演に行ったか）。「user_id ＋ event_id」で一意

RLS は `docs/supabase_init.sql` のほか、**`docs/migration_add_artist_name.sql`**（`events.artist_name` 追加）、**`docs/migration_events_update_policy.sql`**（公演更新ポリシー）を実行済みなら編集も有効。

## 7. ローカルで動かすとき

```bash
cd live-record-app
npm install
npm run dev
```

`.env.local` に `NEXT_PUBLIC_SUPABASE_URL` と `NEXT_PUBLIC_SUPABASE_ANON_KEY` を入れておく。  
http://localhost:3000 でトップ → ログイン → 記録する → マイ記録 → ランキング、まで確認できる。

## 8. 画面で確認する手順（どこまで進んでるか）

開発サーバーを起動した状態で、ブラウザで次の順に開いてみてください。

| 順番 | 開くURL | 確認すること |
|------|---------|----------------|
| 1 | http://localhost:3000 | **トップ**。「ライブ記録アプリ」と「ログイン」ボタンが出ていれば OK。 |
| 2 | http://localhost:3000/login | **ログイン画面**。メール・パスワード欄と「ログイン」「新規登録」ボタン。ここでアカウント作成 or ログイン。 |
| 3 | http://localhost:3000 | ログイン後、もう一度トップ。今度は「記録する」「マイ記録」「ランキング」のリンクと「ログアウト」が出ていれば OK。 |
| 4 | http://localhost:3000/record | **記録する**。会場・公演日・**公演名**・**アーティスト名**・メモの入力欄。 |
| 5 | 記録を1件保存 | 会場を選ぶ（or 新規登録）→ 日付・公演名・アーティスト名を入れて「記録する」→ 保存後 **マイ記録** に飛べば OK。 |
| 6 | http://localhost:3000/my | **マイ記録**。最近10件の記録＋「カレンダーで見る」「自分だけのランキング」ボタン。全件は「全件一覧」または「全件を見る」から。 |
| 7 | 一覧の「編集」をクリック | **公演の編集**画面。公演名・アーティスト名の打ち間違いを直して保存できる。 |
| 8 | http://localhost:3000/profile | **ニックネーム設定**。ランキングに表示する名前を設定。 |
| 9 | http://localhost:3000/ranking | **ランキング**。公演数・会場数・都道府県数のトップ20（ニックネーム表示）。 |

**まとめ:** 1〜9 まで問題なく表示・動作していれば、今日の範囲まで完了です。
