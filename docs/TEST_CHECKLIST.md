# 主要フロー 動作確認チェックリスト（6.1）

本番リリース前に、次の流れを**上から順に**手で触って確認してください。  
チェックしたら `[x]` にすると進捗が分かります。

---

## 事前準備

- [x] `.env.local` に `NEXT_PUBLIC_SUPABASE_URL` と `NEXT_PUBLIC_SUPABASE_ANON_KEY` が入っている
- [x] Supabase で `docs/supabase_init.sql` および必要なマイグレーション（`migration_venues_address.sql`、`migration_venues_update_policy.sql`、**`migration_venues_position_approximate.sql`** など）を実行済み
- [x] `npm run dev` でローカル起動し、ブラウザで http://localhost:3000 を開ける

---

## 1. 認証

- [x] 未ログインでトップを開くと、ログインや記録への導線が分かる
- [x] サインアップ（新規登録）できる
- [x] ログインできる
- [x] ログアウトできる
- [x] ログイン後、トップに「マイ記録」や「記録する」などが表示される

---

## 2. 記録する

- [x] 「記録する」から会場を検索できる
- [x] 会場を選んで公演日・公演名・アーティスト名を入力し、保存できる
- [x] 会場が一覧にない場合、新規会場（名前・都道府県・市区町村・番地）を登録して保存できる
- [x] フェス・対バンで「＋ アーティストを追加」し、複数アーティストで保存できる
- [x] 保存後、マイ記録や全件一覧にその公演が表示される

---

## 3. マイ記録・一覧

- [x] マイ記録トップ（`/my`）で最近の記録が表示される
- [x] 「全件一覧」で全件表示される
- [x] 全件一覧で**絞り込み**（日付順・年・アーティスト・会場のプルダウン）が動く
- [x] 一覧の「編集」から編集画面に飛べる

---

## 4. 編集

- [x] 編集画面で公演名・アーティスト名・メモを変更して保存できる
- [x] 会場の住所を変更して「会場を保存」できる（ポリシー実行済みの場合）

---

## 5. マップ

- [x] マイ記録から「マップで見る」でマップ画面（`/my/map`）が開く
- [x] 緯度経度が入っている会場がピンで表示される
- [x] ピンをタップすると会場名とライブ一覧が表示される
- [x] 住所はあるが位置未取得の会場が「〇件の会場には住所が…」とリスト表示される
- [x] 「住所から位置を取得する」を押すとジオコーディングが走り、成功分はピンが増える（または概略位置でピンが出る）
- [x] 概略位置の会場がある場合、「正確な位置ではない可能性があります」の注意書きが出る

---

## 6. ランキング

- [x] ランキング画面で公演数・会場数・都道府県数が表示される
- [x] アーティスト別・会場別のランキングが表示される
- [x] `/ranking/period` で月別・年別のランキングが表示される
- [x] 会場やアーティストを選ぶと、その会場／アーティストに何回行ったかのユーザーランキングが表示される

---

## 7. その他

- [x] プロフィール（`/profile`）でニックネーム・実績・バッジが表示される
- [x] シェア用カード（画像出力）が動作する（任意）
- [ ] スマホ表示で崩れや操作しづらいところがないか確認する（6.2）
